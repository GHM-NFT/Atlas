# v20 UX Approval Guide

Use this prototype as the single review surface for the next iteration.

## Recommended review sequence

1. Open **Overview** and judge hierarchy, density and colour consistency.
2. Open each workstream command chamber and review all four tabs.
3. Open **Visual map**, select a workstream, then select a dependency line.
4. Use **Critical path** and confirm that the programme pressure points are understandable.
5. Open **Timeline** and filter by workstream and status.
6. Open **Register**, search, sort, filter, paginate and inspect work items.
7. Add one prototype item and confirm that all required fields are collected.
8. Review at desktop, tablet and mobile widths.

## Approval questions

- Can the most important issue be found in under ten seconds?
- Is status visually distinct from risk?
- Does every action produce a clear result?
- Is any space visually empty without an intentional explanation?
- Does the map clarify dependencies rather than decorate them?
- Does the command chamber provide enough context without becoming a spreadsheet?
- Are colours and terminology consistent across every view?
- Can the approved interface accept live-sheet data without changing the layout?

## Change-request format

For quicker future iterations, record each change as:

`View → component → current problem → desired behaviour → priority`

Example:

`Visual map → dependency label → overlaps the WS003 node → move label to the curve midpoint → high`
